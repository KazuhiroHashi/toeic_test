# TOEIC 総合問題集

TOEIC L&R 形式のオリジナル問題を、ブラウザだけで練習できる静的Webアプリ。インストール・ビルド不要で、スマホでも動きます。

## 収録内容(計108問)

- Part 2 応答問題:25問=本番と同じ問題数(前半15問は標準タイプ、16問目以降は近年増加中の間接応答・変化球タイプ。質問と応答は本番同様、別話者・男女別の声)
- Part 3 会話問題:5会話 × 3問 = 15問(リスニング。本番同様、男女2人の会話+3人の会話を収録)
- Part 4 説明文問題:4トーク × 3問 = 12問(リスニング)
- Part 5 短文穴埋め:30問(本番と同じ問題数。基礎〜上級ミックスで、語彙問題約3割+品詞・動詞形・前置詞/接続詞・仮定法倒置・分詞構文など本番の出題比率を反映)
- Part 6 長文穴埋め:3文書 × 4問 = 12問(文挿入問題を含む)
- Part 7 読解問題:4文書 8問+ダブルパッセージ(クロスリファレンス問題・意図問題・語彙問題を含む)
- ミニ模試:上記から約25問をランダム出題

## ロードマップ:本番と同じ200問構成へ

最終目標は、実際のTOEIC L&Rと同じ構成のフル模試(リスニング100問+リーディング100問、計200問)。現状と目標は以下の通り。

1. Part 1 写真描写:目標 6問(現状 0問。写真素材の用意が必要)
2. Part 2 応答問題:目標 25問(現状 25問 ✅ 本番と同数。間接応答タイプを含む)
3. Part 3 会話問題:目標 13会話39問(現状 5会話15問 → あと8会話。図表問題・意図問題も追加予定)
4. Part 4 説明文問題:目標 10トーク30問(現状 4トーク12問 → あと6トーク。図表問題も追加予定)
5. Part 5 短文穴埋め:目標 30問(現状 30問 ✅ 本番と同数)
6. Part 6 長文穴埋め:目標 4文書16問(現状 3文書12問 → あと1文書)
7. Part 7 読解問題:目標 54問(シングル29問+ダブル10問+トリプル15問。現状 14問 → あと40問)

200問そろった時点で、本番と同じ2時間(リスニング約45分+リーディング75分)のフル模試モードを追加する。

### 音声ファイルの同梱(検討中)

端末やブラウザに依存せず全員が同じ音で聞けるよう、リスニング音声を録音ファイル(mp3)としてアプリに同梱する構想がある。高品質なニューラル音声(米・英・豪・加の男女8名+ナレーター)での生成を予定しているが、生成にはTTSサービスへのネットワークアクセスが必要(現在の開発環境ではブロックされているため未実施)。同梱後は、音声ファイルがある問題は録音を再生し、無い問題は従来どおり音声合成にフォールバックする設計とする。

全問題に日本語の解説と全訳付き。

### 音声(リスニング)の仕組み

ブラウザの音声合成(Web Speech API)で読み上げます。

- 既定では、端末の英語音声のうち**発音の評判が良い声だけ**を自動選抜して使います(iOS の高品質版 Ava/Zoe/Evan/Nathan/Samantha(Enhanced)、英 Daniel/Kate/Serena、豪 Karen/Matilda など)。高品質版(Enhanced/Premium/Natural)があれば最優先します。
- 機械的な旧式ボイス(Fred/Kathy 等)とジョーク用ボイス(Grandpa/Whisper 等)は除外済み。
- **本番と同じ米・英・豪・加の4カ国を問題ごとにローテーション**して配役します(その国の声が端末に無ければ品質上位で代用)。
- ホームの**「声の試聴と選択」画面**で、端末の英語音声を名前付きで試聴し、出題に使う声をチェックボックスで選べます(選択はその端末に保存)。
- 男女の声が見つからない端末では、ピッチ(声の高さ)を大きく変えて話者を区別します。男性の声はこもって聞こえやすいため、やや高めのピッチに調整しています。
- iPhone は「設定 → アクセシビリティ → 読み上げコンテンツ → 声 → 英語」で高品質版の声を追加ダウンロードできます(Ava(Premium)、Evan(Enhanced)、Zoe(Premium)推奨)。追加すると本アプリでも自動的に使われます。
- 本番同様、ナレーターが「Questions 1 through 3 refer to the following conversation.」という導入文を読んでから会話が始まります。
- 3人の会話では2人目の女性に別の声(または別のピッチ)を割り当てます。
- iOS Safari で複数の音声を連続再生すると途中で止まる既知の問題があるため、1文ずつ順番に再生する方式にしています。

## 使い方

### ローカルで開く

`index.html` をブラウザで開くだけで動きます(サーバー不要)。

### GitHub Pages で公開する

1. リポジトリの Settings → Pages を開く
2. Source を「Deploy from a branch」、Branch を `main` / `(root)` にして Save
3. 数分後に `https://<ユーザー名>.github.io/toeic_test/` で公開されます

## 機能

- パート別練習+ミニ模試モード
- 問題を選んで解くモード(パートごとの問題一覧から好きな問題だけを選択して挑戦。結果画面から同じ選択で再挑戦可能。選択モードの成績は自己ベストには含めない)
- 解答直後に正誤・解説・訳を表示
- リスニングはスクリプト非表示で出題 → 解答後にスクリプトと全訳を公開
- Part 2 は本番同様、選択肢を見せずに音声のみで出題
- 結果画面で間違えた問題だけを一覧復習
- パートごとの自己ベスト・挑戦回数を保存(ブラウザの localStorage、端末内のみ)

## 問題の追加方法

問題データは `data/part2.js`〜`data/part7.js` に分かれています。各ファイル内のオブジェクトを同じ形式でコピーして追記するだけで、アプリに自動的に反映されます。`answer` は選択肢配列の添字(0 = A)です。

作問ルール(Part 3/4 など1つの音声に複数設問が付く場合):**ある設問の答えが、他の設問の文や選択肢から推測できてはならない**。例えば Q1 が「どこにいる?」(答え:ホテル)なら、Q2・Q3 の文・選択肢に hotel / reservation などの語を入れない。場所・目的・主題を問う設問と、それを前提にした語を含む設問を同じセットに共存させないこと。
